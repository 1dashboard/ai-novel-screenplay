# 剧本输出质量 — 人工评估 Checklist

用于对 AI 生成的剧本 YAML 进行逐项审查，配合 `python -m src.evaluator` 自动评估使用。

## A. 角色评估

- [ ] **角色遗漏**：原著中的重要角色是否全部出现在角色表中？
- [ ] **角色冗余**：是否有 AI 虚构的、原著中不存在的角色？
- [ ] **性格准确**：每个角色的 `traits` 和 `description` 是否与原著一致？
- [ ] **别名完整**：角色的别名、昵称、化名是否都记录在 `aliases` 中？
- [ ] **关系正确**：`relationships` 中的人物关系是否准确？方向是否正确？
- [ ] **主配角分类**：`protagonist` / `supporting` / `minor` 划分是否合理？

## B. 场景评估

- [ ] **情节覆盖**：原著所有关键情节是否都转化为场景？
- [ ] **情节遗漏**：是否有重要的支线或伏笔被忽略？
- [ ] **场景顺序**：场景排列顺序是否与原著叙事顺序一致？
- [ ] **场景切分**：场景粒度是否合理？（过长应拆分，过短应合并）
- [ ] **地点标注**：每个场景的 `location` 是否准确？
- [ ] **时间标注**：`time_of_day`（日/夜/晨/暮）是否正确？
- [ ] **出场角色**：`characters_present` 是否包含该场景中的所有出场人物？

## C. 内容元素评估

- [ ] **对白忠实度**：`dialogue` 文本是否与原著对白一致？
- [ ] **对白归属**：每段对白是否归给了正确的角色？
- [ ] **动作描写**：`action` 元素是否准确描述了场景中的动作/环境？
- [ ] **转场标记**：`transition` 使用是否恰当？（CUT TO / FADE IN 等）
- [ ] **语气标注**：对白的 `delivery` 字段（如 "愤怒地"、"低声"）是否合适？

## D. 备注 (note) 复核

- [ ] **高严重度备注**：所有 `severity: warning` 的备注是否已处理？
- [ ] **信息性备注**：`severity: info` 的备注是否需要人工决策？
- [ ] **备注遗漏**：AI 不确定但未标注的地方是否也应加备注？

## E. 整体质量

- [ ] **幻觉检查**：是否有原著中完全不存在的情节/对话/角色？
- [ ] **文风一致**：剧本整体语言风格是否与原著的基调一致？
- [ ] **可拍摄性**：场景描述是否具体、可执行？（而非抽象心理描写）
- [ ] **节奏感**：对白与动作的比例是否适合影视呈现？
- [ ] **时长估算**：当前场景数量是否大致对应目标时长？（通常 1 场景 ≈ 2-4 分钟）

## 评估结论

- **可直接使用**：所有关键项通过，仅需微调
- **需局部修改**：部分场景/角色需手动修正，整体框架可用
- **需重新生成**：大量错误或遗漏，建议调整 Prompt 后重新转换

---

> 建议评估流程：先跑 `python -m src.evaluator output.yaml` 看自动评分 → 对着此 Checklist 逐项过 → 在 YAML 中直接修改 → 用 `python main.py validate output.yaml` 确认修改后仍然合法。
